4. Configuración de Cloudinary

  1. Crea una cuenta en cloudinary.com
  2. En el Dashboard, copia: Cloud Name, API Key, API Secret
  3. Ve a Settings → Upload → "Add upload preset"
  4. Configura el preset:
    • Name: bandsocial_preset
    • Signing Mode: Unsigned ← imprescindible
    • Folder: bandsocial | Formats: jpg, png, gif, webp | Max: 10 MB

5. Configuración de EmailJS

  1. Crea una cuenta en emailjs.com
  2. Dashboard → Email Services → "Add New Service" → Gmail
  3. Conecta tu cuenta de Gmail y copia el Service ID
  4. Dashboard → Email Templates → "Create New Template"
    • Template ID: template_contacto
  5. Account → API Keys → copia la Public Key

6. Variables de Entorno

bash# Crea el .env desde la plantilla copy .env.example .env # Windows cp .env.example .env # Mac / Linux

Edita .env con los valores obtenidos en los pasos anteriores:

env# Firebase VITE_FIREBASE_API_KEY=AIzaXXXXXX VITE_FIREBASE_AUTH_DOMAIN=tu-proyecto.firebaseapp.com VITE_FIREBASE_PROJECT_ID=tu-proyecto VITE_FIREBASE_STORAGE_BUCKET=tu-proyecto.appspot.com VITE_FIREBASE_MESSAGING_SENDER_ID=123456789 VITE_FIREBASE_APP_ID=1:123456789:web:abcdef # Cloudinary VITE_CLOUDINARY_CLOUD_NAME=tu-cloud-name VITE_CLOUDINARY_UPLOAD_PRESET=bandsocial_preset # EmailJS VITE_EMAILJS_PUBLIC_KEY=tu-public-key VITE_EMAILJS_SERVICE_ID=service_xxxxxxx VITE_EMAILJS_TEMPLATE_CONTACTO=template_contacto # Admin (UIDs separados por coma) VITE_ADMIN_UIDS=uid1,uid2

⚠️ Nunca subas .env al repositorio. Ya está incluido en .gitignore. Usa .env.example como plantilla para otros colaboradores.

7. Ejecutar el Proyecto

bashnpm run dev # Servidor de desarrollo → localhost:5173 npm run build # Build de producción → /dist npm run preview # Preview del build local npm run test # Tests unitarios con Vitest npm run test:e2e # Tests E2E con Cypress npm run lint # Verificar errores ESLint
✅ Al ejecutar npm run dev la app estará disponible en http://localhost:5173 con hot-reload automático.

8. Despliegue en Netlify

Desde la interfaz web

  1. Ve a netlify.com → "Add new site" → "Import existing project"
  2. Conecta con GitHub y selecciona el repositorio BandSocial
  3. Configurar build:
    • Build command: npm run build
    • Publish directory: dist
  4. En Site configuration → Environment variables agrega todas las variables del .env
  5. Clic en "Deploy site"
📸 Captura

Variables de entorno configuradas en Netlify

Post-despliegue (obligatorio)

  • Firebase Auth: Authentication → Settings → Authorized domains → agrega tu dominio de Netlify
  • Cloudinary CORS: Settings → Security → Allowed fetch domains → agrega tu dominio
🌐 La URL de producción actual es: https://bandsociall.netlify.app

9. Problemas Comunes

ProblemaCausa probableSolución
npm install falla Conflicto de dependencias npm install --legacy-peer-deps
Vite no inicia node_modules corrupto Borra node_modules y ejecuta npm install de nuevo
Firebase: invalid-api-key Variables de entorno incorrectas Revisa el .env y reinicia el servidor
Cloudinary devuelve 401 Preset en modo Signed Cambia el preset a Unsigned en Cloudinary
EmailJS error 403 Dominio no autorizado Agrega tu dominio en EmailJS → Account → Security
Variables undefined en producción No configuradas en Netlify Agrega todas las variables en Netlify y redesplega
